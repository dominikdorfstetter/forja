# Onboarding

After signing in for the first time, Forja walks you through a guided onboarding flow. The process collects a few preferences, helps you create your first site, and gives you a checklist so you know exactly what to do next.

## Initial Survey

The first screen you see after your initial sign-in is a short welcome survey. It asks two questions:

### User Type

Select the option that best describes how you will use Forja:

| Option | Description |
|--------|-------------|
| **Solo creator** | You are an individual managing your own content. |
| **Team** | You work with a small group of collaborators. |
| **Agency** | You manage content for multiple clients or brands. |

### Content Intents

Choose one or more content types you plan to create:

- Blog
- Portfolio
- Marketing site
- Documentation
- Company website

:::tip
These selections help Forja tailor the initial experience -- for example, suggesting relevant modules or templates. You are not locked into these choices. You can enable or disable any feature later in site settings.
:::

## Site Creation Wizard

After the survey, the wizard guides you through creating your first site. Each step collects one piece of configuration:

1. **Site name** -- the display name for your site (e.g., "My Tech Blog").
2. **Slug** -- the URL-safe identifier derived from the name. You can customize it.
3. **Description** -- a brief summary of what the site is about.
4. **Locale** -- the default language for your content (e.g., English, German, French).
5. **Module selection** -- choose which features to enable (blogs, pages, media, navigation, etc.). Modules can be toggled later.
6. **Editorial workflow** (optional) -- enable this if your team uses draft/review/publish stages. Solo creators can skip this.

Once you confirm, Forja creates the site and drops you into the dashboard.

## Setup Checklist

After your first site is created, the dashboard displays a setup checklist. This checklist guides you through the essential steps to get your site ready:

| Checklist Item | What It Does |
|----------------|-------------|
| **Configure locales** | Add additional languages if your site serves multilingual content. |
| **Create your first page** | Opens the page editor so you can build a landing page or about page. |
| **Create your first blog post** | Opens the blog editor with a blank post. |
| **Set up navigation** | Opens the navigation builder to define your site's menu structure. |
| **Publish content** | Guides you to publish a draft once you have content ready. |
| **Seed sample content** (optional) | Populates the site with example blog posts and pages you can edit or delete. |
| **Invite team members** (optional) | Opens the members page to invite collaborators. Only shown for team/agency setups. |

### How the Checklist Works

- Each item is a direct link -- click it to navigate to the relevant feature.
- Completed items show a checkmark and move to the bottom of the list.
- Progress is tracked per-site, so each site you create gets its own checklist.
- The checklist disappears from the dashboard once all required items are complete, or you can dismiss it manually.

:::note
The optional items (seed content, invite members) are not required to complete the checklist. They appear based on the user type you selected during the initial survey.
:::
